# s14: The ReAct Pattern (ReAct推理行动模式)

`s01 > s02 > s03 > s04 > s05 > s06 > s07 > s08 > s09 > [ s10 ] > s11 > s12`

> *"Think before you act, observe before you think again"* -- 先思考再行动，观察之后再思考。

## 问题

标准的 `tool_use` 模式把思考过程藏在结构化响应里 -- LLM 的推理链对开发者不可见。更关键的是，并非所有 LLM API 都支持 tool_use，如果模型只能输出纯文本，整个工具调用机制就失效了。我们需要一种**不依赖 API 特定能力**的工具调用方式，同时让推理过程透明可见。

## 解决方案

```
            REACT_ENABLE = true                 REACT_ENABLE = false
          ┌──────────────────┐               ┌──────────────────┐
          │  不发送 tools     │               │  发送 tools      │
          │  给 LLM API      │               │  给 LLM API      │
          └────────┬─────────┘               └────────┬─────────┘
                   │                                  │
                   ▼                                  ▼
          LLM 输出纯文本:                     LLM 输出结构化:
          ┌──────────────────┐               ┌──────────────────┐
          │ Thought: 分析..  │               │ ToolUseBlock {   │
          │ Action: bash     │               │   name: "bash"   │
          │ Action Input: {} │               │   input: {...}   │
          └────────┬─────────┘               │ }                │
                   │                          └────────┬─────────┘
                   ▼                                   │
          正则表达式解析:                               │
          ┌──────────────────┐                        │
          │ Pattern.compile  │                        │
          │ ("Action:...")   │                        │
          │ ("Action Input..")│                        │
          └────────┬─────────┘                        │
                   │                                  │
                   └──────────┬───────────────────────┘
                              ▼
                     invokeTool()
                              │
                              ▼
                     工具执行结果
                              │
               ┌──────────────┴──────────────┐
               ▼                             ▼
     "Observation: ..."             ToolResultBlockParam
     (注入为 USER 消息)              (追加为 tool_result)
               │                             │
               └──────────────┬──────────────┘
                              ▼
                     继续循环 (直到无 Action)
```

一个 `REACT_ENABLE` 开关控制两条完全不同的执行路径。ReAct 模式下工具不发给 API，LLM 以纯文本输出 Thought/Action/Action Input，ZQAgent 用正则解析后执行工具，再将 Observation 注入为用户消息。

## 工作原理

### 1. 双模式入口 -- Agent14

Agent14 是入口类，根据 `REACT_ENABLE` 选择不同的系统提示词和执行模式：

```java
public class Agent14 {
    public static void main(String[] args) {
        AnthropicClient client = AnthropicOkHttpClient.builder()
                .apiKey(API_KEY).baseUrl(BASE_URL).build();

        List<ToolDefinition> tools = new ArrayList<>();
        tools.add(BashDefinition);
        tools.add(EditFileDefinition);
        tools.add(WriteFileDefinition);
        tools.add(ReadFileDefinition);
        tools.add(ListFilesDefinition);
        tools.add(ContentSearchDefinition);

        ZQAgent agent = new ZQAgent(client, MODEL, tools);

        String systemPrompt;
        if (REACT_ENABLE) {
            systemPrompt = buildReActSystemPrompt(tools);
        } else {
            systemPrompt = buildNormalSystemPrompt(tools);
        }
        agent.setSystemPrompt(systemPrompt);
        agent.run();
    }
}
```

**代码解释**:
- `REACT_ENABLE`: 来自 `AIConstants` 的布尔开关，决定整个执行模式
- `buildReActSystemPrompt(tools)`: 构建 ReAct 模式提示词，强制 Thought/Action/Action Input/Observation 格式
- `buildNormalSystemPrompt(tools)`: 构建标准模式提示词，列出工具描述
- 组合模式：创建 `ZQAgent` 实例而非继承，与 Agent01、Agent02、Agent13 一致

### 2. ReAct 系统提示词 -- buildReActSystemPrompt()

ReAct 提示词严格约束 LLM 的输出格式，确保正则解析能正确工作：

```java
private static String buildReActSystemPrompt(List<ToolDefinition> tools) {
    StringBuilder sb = new StringBuilder();
    sb.append("你是一个使用 ReAct 模式的 AI 助手。ReAct 模式结合了推理和行动。\n\n");
    sb.append("当需要使用工具时，请按照以下格式思考和行动：\n\n");
    sb.append("Thought: 思考并确定下一步的最佳行动方案\n");
    sb.append("Action: 工具名称\n");
    sb.append("Action Input: 工具参数（JSON 对象）\n");
    sb.append("Observation: 工具执行结果\n\n");
    sb.append("重要规则：\n");
    sb.append("- 不使用工具时，回复中不要出现 Thought、Action、Action Input\n");
    sb.append("- 使用工具前，检查是否缺少必要参数\n");
    sb.append("- 一次只能返回一个 Thought/Action/Action Input/Observation\n");
    sb.append("- 不能私自给 Observation 赋值\n\n");
    sb.append("可用工具：\n");
    for (ToolDefinition tool : tools) {
        sb.append("- ").append(tool.getName())
          .append(": ").append(tool.getDescription()).append("\n");
    }
    return sb.toString();
}
```

**代码解释**:
- 四元素格式：Thought（思考）/ Action（行动）/ Action Input（参数）/ Observation（结果）
- 严格约束：一次只返回一组，不私自赋值 Observation，缺参数时向用户提问
- 动态工具列表：遍历 `tools` 生成可用工具描述，LLM 据此选择 Action

### 3. ReAct 模式下不发送工具 -- chatMessage()

这是 ReAct 模式的关键：**工具不发送给 API**。LLM 只看到提示词中的工具列表，以纯文本输出 Action：

```java
protected Message chatMessage(List<MessageParam> messageParams) {
    List<ToolUnion> anthropicTools = new ArrayList<>();
    for (ToolDefinition tool : tools) {
        anthropicTools.add(ToolUnion.ofTool(
                Tool.builder()
                        .name(tool.getName())
                        .description(tool.getDescription())
                        .inputSchema(tool.getInputSchema())
                        .build()
        ));
    }

    MessageCreateParams.Builder messageBuilder = MessageCreateParams.builder()
            .model(model)
            .messages(messageParams);

    if (!REACT_ENABLE && tools != null && !tools.isEmpty()) {
        messageBuilder.tools(anthropicTools);
    }

    if (systemPrompt != null && !systemPrompt.isEmpty()) {
        messageBuilder.system(systemPrompt);
    }
    messageBuilder.maxTokens(MAX_TOKENS);
    messageBuilder.temperature(TEMPERATURE);

    return client.messages().create(messageBuilder.build());
}
```

**代码解释**:
- `!REACT_ENABLE`: 只有标准模式才发送工具给 API
- ReAct 模式下 `messageBuilder.tools()` 不被调用，API 只收到纯文本请求
- LLM 通过系统提示词了解可用工具，以纯文本格式输出 Action

### 4. 正则解析引擎 -- run()

ReAct 模式的核心逻辑在 `ZQAgent.run()` 中，用正则从 LLM 文本输出中提取工具调用信息：

```java
if (REACT_ENABLE) {
    String resultText = "";
    for (ContentBlock content : message.content()) {
        if (content.isText()) {
            resultText = content.text().map(TextBlock::text).orElse("");
        }
    }

    messageParams.add(MessageParam.builder()
            .role(MessageParam.Role.ASSISTANT)
            .content(resultText).build());

    if (resultText.contains("Action:")) {
        Pattern actionPattern = Pattern.compile("Action:\\s*([^\\n]+)");
        Pattern inputPattern = Pattern.compile(
            "Action Input:\\s*(\\{.*?\\})(?=\\s*\n|$)", Pattern.DOTALL);

        Matcher actionMatcher = actionPattern.matcher(resultText);
        Matcher inputMatcher = inputPattern.matcher(resultText);

        if (actionMatcher.find()) {
            String targetToolName = actionMatcher.group(1).trim();
            String inputJson = inputMatcher.find() ? inputMatcher.group(1) : "{}";

            ToolDefinition targetTool = null;
            for (ToolDefinition tool : tools) {
                if (tool.getName().equals(targetToolName)) {
                    targetTool = tool;
                    break;
                }
            }

            String observation = "";
            if (targetTool != null) {
                observation = invokeTool(targetTool, inputJson);
            } else {
                observation = "错误: 未找到工具 " + targetToolName;
            }

            messageParams.add(MessageParam.builder()
                    .role(MessageParam.Role.USER)
                    .content("Observation: " + observation).build());

            message = chatMessage(messageParams);
            messageParams.add(message.toParam());
        }
    } else {
        System.out.printf("AI: %s%n", resultText);
        break;
    }
} else {
    // 标准 tool_use 流程
}
```

**代码解释**:
- `resultText.contains("Action:")`: 检测 LLM 是否请求调用工具
- `Pattern.compile("Action:\\s*([^\\n]+)")`: 提取 Action 后面的工具名称
- `Pattern.compile("Action Input:\\s*(\\{.*?\\})(?=\\s*\\n|$)", Pattern.DOTALL)`: 提取 JSON 格式的参数，`DOTALL` 支持跨行匹配
- `invokeTool(targetTool, inputJson)`: 直接调用工具执行
- `"Observation: "` 前缀：将结果注入为用户消息，LLM 据此继续推理
- 无 `Action:` 则视为最终答案，`break` 退出内层循环

### 5. ReActInput 数据模型

`ReActInput` 定义了 ReAct 模式的四个核心字段，但**不作为工具注册**：

```java
public class ReActInput {
    @JsonProperty("thought")
    private String thought;

    @JsonProperty("action")
    private String action;

    @JsonProperty("actionInput")
    private String actionInput;

    @JsonProperty("observation")
    private String observation;
}
```

**代码解释**:
- 四个字段对应 ReAct 的 Thought / Action / Action Input / Observation
- `@JsonProperty` 注解用于 JSON 序列化/反序列化
- 这个类仅作为数据模型存在，未在 `ToolDefinition` 中注册为工具
- ReAct 是**执行模式**，不是**可调用工具**

### 6. ZQAgent 本地副本

Agent14 拥有自己的 `ZQAgent` 副本，与标准版本的关键差异：

| 方法 | 标准 ZQAgent | Agent14 的 ZQAgent |
|------|-------------|-------------------|
| `chatMessage()` | 始终发送 tools | `REACT_ENABLE` 为 false 时才发送 |
| `run()` | 仅处理 tool_use block | 额外处理 ReAct 文本解析分支 |
| `invokeTool()` | 仅处理 TypedToolInvoker | 额外支持 `Function<String,String>` 模式（MCP 兼容） |

**代码解释**:
- 本地副本确保 ReAct 分支不影响其他模块的标准行为
- `invokeTool()` 的 MCP 兼容分支：当 `tool.getType() == null` 时，将输入包装为 `{"tool_name": ..., "input": ...}` 传入 `Function<String,String>`

## 工具数量表

| 工具名 | 类型 | 输入参数 | 说明 |
|--------|------|----------|------|
| bash | Function | command, type | 执行 Shell 命令（cmd/powershell/bash） |
| read_file | Function | path | 读取文件内容 |
| write_file | Function | path, content | 写入文件 |
| edit_file | Function | path, oldStr, newStr | 编辑文件（字符串替换） |
| list_files | Function | path, fileType | 列出目录文件 |
| content_search | Function | pattern, path, fileType, caseSensitive | ripgrep 搜索文件内容 |

**总计：6 个工具**（与 Agent01-08 相同的基础工具集）

> 注：ReAct 不是工具，而是执行模式。`ReActInput` 仅作为数据模型存在。

## 相对于 s01 的变更

| 组件 | s01 (Agent Loop) | s14 (ReAct Pattern) | 变更说明 |
|------|-----------------|---------------------|----------|
| 架构模式 | 组合模式 | 组合模式 | 不变 |
| ZQAgent | 标准版本 | **本地副本** | 新增 `REACT_ENABLE` 分支 |
| chatMessage() | 始终发送 tools | 条件发送 | `!REACT_ENABLE` 时才发送 |
| run() | tool_use block 解析 | **正则文本解析** | ReAct 模式用正则提取 Action |
| 工具结果注入 | ToolResultBlockParam | **Observation 用户消息** | `role=USER` + `"Observation: "` 前缀 |
| 系统提示词 | 单一提示词 | **双提示词** | `buildReActSystemPrompt()` + `buildNormalSystemPrompt()` |
| 工具数量 | 1 (bash) | 6 | 增加了文件操作和搜索工具 |
| invokeTool() | TypedToolInvoker | **双模式** | 额外支持 `Function<String,String>` |
| 推理可见性 | 不可见 | **完全可见** | Thought 过程以纯文本输出 |
| API 兼容性 | 依赖 tool_use | **不依赖** | ReAct 模式仅需文本生成能力 |
