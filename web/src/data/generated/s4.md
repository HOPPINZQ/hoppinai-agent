# s04: Subagents（子智能体）

`s01 > s02 > s03 > [ s04 ] s05 > s06 | s07 > s08 > s09 > s10 > s11 > s12`

> *"大任务拆小，每个小任务干净的上下文"* -- 子智能体使用独立的 messages[]，不污染主对话。

## 问题

智能体工作越久，messages 数组越胖。每次读文件、跑命令的输出都永久留在上下文里。"这个项目用什么测试框架？" 可能要读 5 个文件，但父智能体只需要一个词："Maven + JUnit。"

## 解决方案

```
Parent agent                     SubAgent (static utility)
+------------------+             +------------------+
| messages=[...]   |             | messages=[]      | <-- fresh
|                  |  dispatch   |                  |
| tool: sub_agent  | ----------> | while tool_use:  |
|   prompt="..."   |             |   call tools     |
|                  |  summary    |   append results |
|   result = "..." | <---------- | return last text |
+------------------+             +------------------+

Parent context stays clean. SubAgent context is discarded.
```

## 工作原理

### 1. SubAgent -- 静态工具类

子智能体不是 Agent04 的实例方法，而是 `com.hoppinzq.agent.base.SubAgent` 中的一个**静态方法**：

```java
public class SubAgent {
    public static String executeSubAgent(AnthropicClient client,
                                         String model,
                                         SubAgentInput input) {
        // 创建子智能体工具列表（不含 sub_agent，防止递归）
        List<ToolDefinition> subTools = new ArrayList<>();
        subTools.add(ReadFileDefinition);
        subTools.add(EditFileDefinition);
        subTools.add(WriteFileDefinition);

        // 创建子智能体实例（独立上下文）
        ZQAgent subAgent = new ZQAgent(client, model, subTools);
        subAgent.setSystemPrompt("你是一个子智能体...");

        // 子智能体运行自己的循环，只返回最终文本
        String result = subAgent.runTask(input.getPrompt());
        return result;
    }
}
```

**要点**：
- `SubAgent` 在 `base` 包下，不是 `tool` 包
- `executeSubAgent()` 是静态方法，不依赖任何实例状态
- 子智能体通过 `new ZQAgent()` 创建，**组合模式**，不复用父类

### 2. 父子智能体工具分离

```
父智能体 (8 个工具)              子智能体 (3 个工具)
├── bash                         ├── read_file
├── read_file                    ├── edit_file
├── edit_file                    └── write_file
├── write_file
├── list_files
├── content_search
├── todo
└── sub_agent        ← 子智能体没有此工具，防止递归生成
```

子智能体只有文件操作工具，**没有** bash（安全考虑）、没有 sub_agent（防止递归）、没有 todo（子任务不需要任务管理）、没有 list_files 和 content_search。

### 3. sub_agent 工具定义

在 ToolDefinition.java 中注册：

```java
public static ToolDefinition SubAgentDefinition = new ToolDefinition(
    "sub_agent",
    "将子任务委托给子智能体处理。子智能体拥有读取文件、写入文件、"
    + "编辑文件和搜索内容等工具能力...",
    createInputSchema(
        Map.of("prompt", createProperty("string",
            "给子智能体的任务提示词，详细描述需要完成的任务")),
        List.of("prompt")
    ),
    SubAgentInput.class,
    Tools::executeSubAgent    // 静态方法引用
);
```

### 4. 智能文件编辑特性

子智能体的编辑操作实现了空白字符归一化处理，消除 Windows/Unix 换行符和制表符/空格的差异：

```java
private static String normalizeForReplace(String str) {
    String normalized = str.replace("\r\n", "\n").replace("\r", "\n");
    normalized = normalized.replace("\t", "    ");
    return normalized;
}
```
