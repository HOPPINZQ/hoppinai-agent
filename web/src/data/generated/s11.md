# s11: Autonomous Agents（自主智能体）

`s01 > s02 > s03 > s04 > s05 > s06 | s07 > s08 > s09 > s10 > [ s11 ] s12`

> *"Teammates scan the board and claim tasks themselves"* -- no need for the lead to assign each one

## 问题

在 s09-s10 中，队友只在被明确告知时才工作。lead必须为每个任务生成队友并指定提示。10个未认领任务？lead需要手动分配每个任务。无法扩展。

真正的自主性：队友自己扫描任务板，认领未认领的任务，完成它们，然后寻找更多任务。

一个细微之处：在上下文压缩（s06）后，智能体可能忘记自己是谁。身份重新注入解决了这个问题。

## 解决方案

```
队友生命周期与空闲循环：

+-------+
| spawn |
+---+---+
    |
    v
+-------+   tool_use     +-------+
| WORK  | <------------- |  LLM  |
+---+---+                +-------+
    |
    | stop_reason != tool_use (or idle tool called)
    v
+--------+
|  IDLE  |  poll every 5s for up to 60s
+---+----+
    |
    +---> check inbox --> message? ----------> WORK
    |
    +---> scan .tasks/ --> unclaimed? -------> claim -> WORK
    |
    +---> 60s timeout ----------------------> SHUTDOWN

上下文压缩后身份重新注入：
  if len(messages) <= 3:
    messages.insert(0, identity_block)
```

## Java 实现原理

### 1. TaskBoard - 任务板管理器

```java
public class TaskBoard {
    private final Path tasksDir;

    /**
     * 扫描未认领的任务
     * 条件：status=pending, owner=null, blockedBy=null
     */
    public List<Map<String, Object>> scanUnclaimedTasks() {
        List<Map<String, Object>> unclaimed = new ArrayList<>();
        
        for (Path file : taskFiles) {
            JsonNode task = readTask(file);
            
            String status = task.get("status").asText();
            boolean hasOwner = task.has("owner");
            boolean hasBlockedBy = task.has("blockedBy") && 
                                  task.get("blockedBy").size() > 0;
            
            if ("pending".equals(status) && !hasOwner && !hasBlockedBy) {
                unclaimed.add(convertToMap(task));
            }
        }
        return unclaimed;
    }

    /**
     * 认领任务
     */
    public String claimTask(int taskId, String owner) {
        Path taskPath = tasksDir.resolve("task_" + taskId + ".json");
        
        // 读取任务
        Map<String, Object> task = readTask(taskPath);
        
        // 更新owner和status
        task.put("owner", owner);
        task.put("status", "in_progress");
        
        // 写回文件
        writeTask(taskPath, task);
        
        return "Claimed task #" + taskId + " for " + owner;
    }
}
```

### 2. 空闲循环（Idle Polling）

```java
/**
 * 空闲阶段：轮询收件箱和任务板
 */
private boolean idlePoll(String name, List<MessageParam> messages) {
    int polls = IDLE_TIMEOUT / POLL_INTERVAL;  // 60 / 5 = 12
    
    for (int i = 0; i < polls; i++) {
        try {
            Thread.sleep(POLL_INTERVAL * 1000);
        } catch (InterruptedException e) {
            break;
        }
        
        // 检查收件箱
        List<Map<String, Object>> inbox = messageBus.readInbox(name);
        if (!inbox.isEmpty()) {
            for (Map<String, Object> msg : inbox) {
                if ("shutdown_request".equals(msg.get("type"))) {
                    return false;  // 退出
                }
                messages.add(createUserMessage(msg));
            }
            return true;  // 恢复工作
        }
        
        // 扫描任务板
        List<Map<String, Object>> unclaimed = taskBoard.scanUnclaimedTasks();
        if (!unclaimed.isEmpty()) {
            Map<String, Object> task = unclaimed.get(0);
            
            // 认领任务
            taskBoard.claimTask((Integer) task.get("id"), name);
            
            // 身份重新注入
            if (messages.size() <= 3) {
                messages.add(0, createIdentityBlock(name, role, teamName));
            }
            
            // 添加任务提示
            messages.add(createUserMessage(
                "<auto-claimed>Task #" + task.get("id") + ": " + task.get("subject") + "</auto-claimed>"
            ));
            
            return true;  // 恢复工作
        }
    }
    
    return false;  // 超时，关闭
}
```

### 3. 身份重新注入

```java
/**
 * 创建身份块
 */
private MessageParam createIdentityBlock(String name, String role, String teamName) {
    String identity = String.format(
        "<identity>You are '%s', role: %s, team: %s. Continue your work.</identity>",
        name, role, teamName
    );
    
    return MessageParam.builder()
        .role(MessageParam.Role.USER)
        .content(identity)
        .build();
}
```

### 4. 主循环

```java
public void autonomousLoop(String name, String role, String prompt) {
    while (true) {
        // -- WORK PHASE --
        for (int i = 0; i < 50; i++) {
            response = client.messages().create(...);
            
            if (response.stopReason() != Message.StopReason.TOOL_USES) {
                break;  // 进入IDLE
            }
            
            for (ContentBlock block : response.content()) {
                if (isIdleTool(block)) {
                    idleRequested = true;
                    break;
                }
                executeTool(block);
            }
            
            if (idleRequested) {
                break;  // 进入IDLE
            }
        }
        
        // -- IDLE PHASE --
        setStatus(name, "idle");
        boolean resume = idlePoll(name, messages);
        
        if (!resume) {
            setStatus(name, "shutdown");
            return;  // 退出
        }
        
        setStatus(name, "working");
    }
}
```

## 核心优势

1. **自主性**：队友自动寻找和认领任务
2. **可扩展**：无需lead手动分配每个任务
3. **高效轮询**：5秒间隔，避免频繁扫描
4. **优雅超时**：60秒无任务自动关闭
5. **身份保持**：上下文压缩后重新注入身份

## 执行示例

**自主认领任务流程**：

```
用户: 在.tasks/创建3个任务
AI:  [任务创建完成]

用户: 生成alice和bob
AI:  [spawn_teammate alice] -> Spawned
    [spawn_teammate bob] -> Spawned

[alice进入IDLE]
alice: [扫描任务板] -> 认领task_1
-> 开始工作

[bob进入IDLE]
bob: [扫描任务板] -> 认领task_2
-> 开始工作

[alice完成task_1]
alice: [进入IDLE] -> 认领task_3
-> 继续工作

[bob完成task_2]
bob: [进入IDLE] -> 无任务
-> 等待60秒后关闭
```

## 相对于 s10 的变更

| 组件 | s10 | s11 |
|-----|-----|-----|
| Tools | 12 | 14 (+idle, +claim_task) |
| Autonomy | Lead-directed | Self-organizing |
| Idle phase | None | Poll inbox + task board |
| Task claiming | Manual only | Auto-claim unclaimed tasks |
| Identity | System prompt only | + re-injection after compress |
| Timeout | None | 60s idle -> auto shutdown |

## 核心特性

- **空闲循环**：WORK -> IDLE -> WORK 循环
- **任务扫描**：查找pending、无owner、无依赖的任务
- **自动认领**：发现任务后自动认领并开始工作
- **身份保持**：压缩后重新注入身份信息
- **超时机制**：60秒无工作自动关闭
