# s06: Context Compact（上下文压缩）

`s01 > s02 > s03 > s04 > s05 > [ s06 ] | s07 > s08 > s09 > s10 > s11 > s12`

> *"上下文总会满，要有办法腾地方"*

## 问题

Agent05 已经具备完整的工具集（9 个工具），但在长时间运行或处理大型项目时，对话历史会不断膨胀，最终耗尽上下文窗口。具体表现为：

- 工具结果不断累积，每次 `read_file` 返回的文件内容全部保留
- 多轮对话后，token 数量逼近 API 限制
- 智能体无法继续工作，或被迫丢失早期上下文

## 解决方案

Agent06 引入 **ContextCompactor** 三层压缩策略，在对话过程中持续控制上下文大小：

```
                    上下文压缩流程
┌──────────────────────────────────────────────────────┐
│                                                      │
│  ┌─────────────────────────────────────────────┐     │
│  │  Layer 1: 微压缩 (microCompact)             │     │
│  │  ├─ 触发: 每次 LLM 调用前 + 每次工具执行后  │     │
│  │  ├─ 方式: 旧 tool_result → "[已执行: xxx]"  │     │
│  │  └─ 保留: 最近 KEEP_RECENT (3) 个完整结果   │     │
│  └──────────────────┬──────────────────────────┘     │
│                     │                                │
│  ┌──────────────────▼──────────────────────────┐     │
│  │  estimateTokens() > TOKEN_THRESHOLD?        │     │
│  │  算法: JSON序列化.length() / 4               │     │
│  └──────────────────┬──────────────────────────┘     │
│              YES    │                                │
│  ┌──────────────────▼──────────────────────────┐     │
│  │  Layer 2: 自动压缩 (autoCompact)             │     │
│  │  ├─ 保存完整对话 → .transcripts/*.jsonl      │     │
│  │  ├─ LLM 生成摘要 (maxTokens=2000)           │     │
│  │  └─ [压缩提示+摘要, 助手确认] 替换原始列表   │     │
│  └──────────────────┬──────────────────────────┘     │
│                     │                                │
│  ┌──────────────────▼──────────────────────────┐     │
│  │  Layer 3: 手动压缩 (compact 工具)            │     │
│  │  ├─ 触发: 用户调用 compact(focus=...)        │     │
│  │  └─ 执行: manualCompactRequested → autoCompact│     │
│  └─────────────────────────────────────────────┘     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## 工作原理

### 1. 双触发点压缩

Agent06 在两个位置都嵌入压缩逻辑，形成双保险：

```java
// 触发点1: chatMessage() - LLM 调用前
@Override
protected Message chatMessage(List<MessageParam> messageParams) {
    // Layer 1: 微压缩（传入副本，不影响原列表）
    List<MessageParam> compactedParams = compactor.microCompact(new ArrayList<>(messageParams));

    // Layer 2: 自动压缩
    if (ContextCompactor.estimateTokens(compactedParams) > TOKEN_THRESHOLD) {
        compactedParams = compactor.autoCompact(compactedParams);
        this.messageParams.clear();
        this.messageParams.addAll(compactedParams);
    }
    return super.chatMessage(compactedParams);
}

// 触发点2: onToolExecution() - 工具执行后
@Override
protected void onToolExecution(List<ContentBlockParam> toolResults) {
    // Todo 提醒 + Layer 1 + Layer 2 + Layer 3
    compactor.microCompact(messageParams);

    if (compactor.estimateTokens(messageParams) > TOKEN_THRESHOLD) {
        List<MessageParam> compressed = compactor.autoCompact(messageParams);
        messageParams.clear();
        messageParams.addAll(compressed);
    }

    if (manualCompactRequested) {
        List<MessageParam> compressed = compactor.autoCompact(messageParams);
        messageParams.clear();
        messageParams.addAll(compressed);
        manualCompactRequested = false;
    }
}
```

### 2. 微压缩实现（microCompact）

```
输入: 10 个工具结果的消息列表
  ├─ [tool_result: bash 完整输出 500行]
  ├─ [tool_result: read_file 完整内容 200行]
  ├─ ... (7 个更多结果)
  └─ [tool_result: edit_file 确认]

microCompact(KEEP_RECENT=3):
  ├─ [tool_result: "[已执行: bash]"]          ← 占位符
  ├─ [tool_result: "[已执行: read_file]"]     ← 占位符
  ├─ ... (4 个更多占位符)
  ├─ [tool_result: 完整内容]                   ← 保留
  ├─ [tool_result: 完整内容]                   ← 保留
  └─ [tool_result: 完整内容]                   ← 保留
```

实现细节：
- 使用 `ToolResultInfo` record 跟踪工具结果位置（消息索引 + 内容块索引）
- 建立 `toolId → toolName` 映射，生成有意义的占位符
- 仅在结果数 > `KEEP_RECENT` 时才执行压缩

### 3. 自动压缩实现（autoCompact）

```
步骤1: 保存对话
  .transcripts/transcript_1710000000000.jsonl
  └─ 每条消息一行 JSON

步骤2: 生成摘要
  截取消息文本（限制 80000 字符）
  → LLM 生成: "已完成工作 / 当前状态 / 关键决策"

步骤3: 构建新消息列表
  [user: "[对话已压缩。完整记录: .transcripts/xxx.jsonl]\n\n摘要..."]
  [assistant: "收到。我已从摘要中获取了上下文。继续工作。"]
```
