# s12: Worktree Task Isolation（工作树任务隔离）

`s01 > s02 > s03 > s04 > s05 > s06 | s07 > s08 > s09 > s10 > s11 > [ s12 ]`

> *"Each works in its own directory, no interference"* -- tasks manage goals, worktrees manage directories, bound by ID

## 问题

到 s11 为止，智能体可以自主认领并完成任务。但所有任务都在一个共享目录中运行。两个智能体同时重构不同模块会冲突：agent A编辑config.py，agent B也编辑config.py，未暂存的更改混合，两者都无法干净回滚。

任务板跟踪*做什么*但不关心*在哪里做*。解决方案：给每个任务自己的git worktree目录。任务管理目标，worktree管理执行上下文。通过任务ID绑定它们。

## 解决方案

```
Control plane (.tasks/)             Execution plane (.worktrees/)
+------------------+                +------------------------+
| task_1.json      |                | auth-refactor/         |
|   status: in_progress  <------>   branch: wt/auth-refactor
|   worktree: "auth-refactor"   |   task_id: 1             |
+------------------+                +------------------------+
| task_2.json      |                | ui-login/              |
|   status: pending    <------>     branch: wt/ui-login
|   worktree: "ui-login"       |   task_id: 2             |
+------------------+                +------------------------+

State machines:
  Task:     pending -> in_progress -> completed
  Worktree: absent  -> active      -> removed | kept
```

## 核心概念

### 1. 双平面架构

**Control Plane（控制平面）**：
- .tasks/task_*.json
- 管理任务目标、状态、依赖
- task_id、subject、status、worktree字段

**Execution Plane（执行平面）**：
- .worktrees/{name}/
- Git worktree目录
- index.json注册表
- events.jsonl事件流

### 2. Task-Worktree绑定

```java
task_1.json {
  "id": 1,
  "subject": "Implement auth refactor",
  "status": "in_progress",
  "worktree": "auth-refactor"  // 绑定到worktree
}

index.json {
  "worktrees": [{
    "name": "auth-refactor",
    "path": ".../.worktrees/auth-refactor",
    "branch": "wt/auth-refactor",
    "task_id": 1,  // 绑定到task
    "status": "active"
  }]
}
```

### 3. 生命周期事件

每个操作都记录到.events.jsonl：

```json
{
  "event": "worktree.create.after",
  "task": {"id": 1},
  "worktree": {"name": "auth-refactor", "status": "active"},
  "ts": 1730000000
}
```

## 工作流程

1. **创建任务**
   ```
   task_create(subject="Implement auth refactor")
   -> .tasks/task_1.json (status=pending, worktree="")
   ```

2. **创建worktree并绑定**
   ```
   worktree_create(name="auth-refactor", task_id=1)
   -> git worktree add -b wt/auth-refactor .worktrees/auth-refactor HEAD
   -> task_1.json: worktree="auth-refactor", status="in_progress"
   -> index.json: 添加auth-refactor条目
   ```

3. **在worktree中运行命令**
   ```
   worktree_run(name="auth-refactor", command="git status --short")
   -> 在.worktrees/auth-refactor/目录中执行
   ```

4. **关闭**
   ```
   worktree_remove(name="auth-refactor", complete_task=true)
   -> 删除worktree目录
   -> task_1.json: status="completed", worktree=""
   -> emit "task.completed" 事件
   ```

## 核心优势

1. **目录隔离**：每个任务独立工作目录，避免冲突
2. **并行执行**：多个任务同时进行，互不干扰
3. **Git集成**：使用git worktree，支持分支管理
4. **可追溯**：完整的事件日志
5. **原子操作**：创建和删除都是原子的

## 相对于 s11 的变更

| 组件 | s11 | s12 |
|-----|-----|-----|
| Coordination | Task board only | Task board + worktree binding |
| Execution scope | Shared directory | Task-scoped isolated directory |
| Recoverability | Task status only | Task status + worktree index |
| Teardown | Task completion | Task completion + keep/remove |
| Lifecycle visibility | Implicit logs | Explicit events in events.jsonl |

## 核心特性

- **双平面架构**：控制平面+执行平面分离
- **Git Worktree**：使用git worktree实现目录隔离
- **任务绑定**：task_id关联任务和工作树
- **事件流**：完整的生命周期事件记录
- **原子操作**：创建和删除工作树的原子性
