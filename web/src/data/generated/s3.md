# s03: TodoWrite (待办写入)

`s01 > s02 > [ s03 ] s04 > s05 > s06 | s07 > s08 > s09 > s10 > s11 > s12`

> *"没有计划的 agent 走哪算哪"* -- 先列步骤再动手, 完成率翻倍。

## 问题

多步任务中, 模型会丢失进度 -- 重复做过的事、跳步、跑偏。对话越长越严重: 工具结果不断填满上下文, 系统提示的影响力逐渐被稀释。一个 10 步重构可能做完 1-3 步就开始即兴发挥, 因为 4-10 步已经被挤出注意力了。

## 解决方案

```
+--------+      +-------+      +---------+
|  User  | ---> |  LLM  | ---> | Tools   |
| prompt |      |       |      | + todo  |
+--------+      +---+---+      +----+----+
                    ^                |
                    |   tool_result  |
                    +----------------+
                          |
              +-----------+-----------+
              | TodoManager state     |
              | [ ] task A            |
              | [>] task B  <- doing  |
              | [x] task C            |
              +-----------------------+
                          |
              if rounds_since_todo >= 3:
                inject <reminder> into toolResult
```

Agent03 是**第一个使用继承模式**的模块 -- `extends ZQAgent` 而非组合。这样做是为了能够 `@Override onToolExecution()` 钩子, 在每次工具执行后检查 todo 是否被更新。

## 工作原理

### 1. 继承模式与钩子函数

Agent03 继承 ZQAgent, 重写 `onToolExecution()` 方法。这是 Agent01/Agent02（组合模式）做不到的:

```java
public class Agent03 extends ZQAgent {
    public static TodoManager todoManager;
    private int roundsSinceTodo = 0;
    private long lastTodoVersion = 0;

    public Agent03(AnthropicClient client, String model,
                   List<ToolDefinition> tools, TodoManager todoManager) {
        super(client, model, tools);
        Agent03.todoManager = todoManager;
        this.lastTodoVersion = todoManager.getVersion();
    }
}
```

**静态字段桥接设计**: `todoManager` 必须是 `static` 的, 因为 `TodoDefinition` 的 handler 引用了 `Agent03.todoManager::updateTodos` -- 静态方法引用要求目标字段也是静态的。

### 2. 工具注册: 7 个工具

```java
List<ToolDefinition> tools = new ArrayList<>();
tools.add(BashDefinition);           // bash 命令执行
tools.add(EditFileDefinition);       // 编辑文件
tools.add(WriteFileDefinition);      // 写入文件
tools.add(ReadFileDefinition);       // 读取文件
tools.add(ListFilesDefinition);      // 列出目录
tools.add(ContentSearchDefinition);  // 搜索内容
tools.add(TodoDefinition);           // 待办事项管理
```

其中 `TodoDefinition` 使用**嵌套 JSON Schema** 定义输入:

```java
public static ToolDefinition TodoDefinition = new ToolDefinition(
    "todo",
    "管理待办事项列表。使用此工具可以添加、更新或完成任务。"
    + "支持任务状态追踪，包括待处理、进行中和已完成三种状态。",
    createInputSchema(
        Map.of("todos",
            Map.of(
                "type", "array",
                "description", "待办事项列表",
                "items", Map.of(
                    "type", "object",
                    "properties", Map.of(
                        "id",      Map.of("type", "string",
                                          "description", "任务的唯一标识符"),
                        "content", Map.of("type", "string",
                                          "description", "待办事项的内容描述"),
                        "status",  Map.of(
                            "type", "string",
                            "enum", List.of("pending", "in_progress", "completed"),
                            "description", "任务状态：pending-待处理, in_progress-进行中, completed-已完成")
                    ),
                    "required", List.of("id", "content", "status")
                )
            )
        ),
        List.of("todos")
    ),
    TodoInput.class,
    Agent03.todoManager::updateTodos
);
```

### 3. TodoManager: 版本追踪与约束

`TodoManager` 使用 `synchronized update()` 方法, 通过 `version` 字段追踪更新次数:

| 约束 | 说明 |
|-----|------|
| 最多 20 项 | `validated.size() > 20` 时抛异常 |
| 同时仅 1 个 in_progress | `inProgressCount > 1` 时抛异常 |
| 内容不能为空 | 每项 `content.trim()` 后检查 |
| 状态三选一 | `pending` / `in_progress` / `completed` |

渲染格式使用状态标记:

```
[ ] 待处理的任务
[>] 进行中的任务 <- 当前正在执行的操作描述
[x] 已完成的任务

(1/3 已完成)
```

### 4. Nag Reminder: 3 轮不更新就追着问

`onToolExecution()` 在每次工具调用后被 ZQAgent 基类触发。它检查 `todoManager.getVersion()` 是否变化:

```java
@Override
protected void onToolExecution(List<ContentBlockParam> toolResults) {
    long currentVersion = todoManager.getVersion();
    if (currentVersion > lastTodoVersion) {
        roundsSinceTodo = 0;
        lastTodoVersion = currentVersion;
    } else {
        roundsSinceTodo++;
    }

    if (roundsSinceTodo >= 3) {
        String reminder = String.format("""
            <reminder>
            您已经 %d 个回合没有更新待办事项列表了。请更新列表以反映当前进度。

            建议操作：
            1. 检查当前任务状态
            2. 更新已完成的任务
            3. 添加新的任务（如果需要）
            4. 调整任务优先级
            </reminder>
            """, roundsSinceTodo);
        toolResults.add(ContentBlockParam.ofText(TextBlockParam.builder()
                .text(reminder).build()));
    }
}
```

关键设计: reminder 不是重新发送系统提示, 而是作为**工具结果的一部分**注入 -- 这样模型在下一轮推理中会看到这个提醒, 但不会占用额外的 API 消息槽位。

## 变更内容

| 组件 | 之前 (s02) | 之后 (s03) |
|-----|-----------|-----------|
| 模式 | 组合 (new ZQAgent) | 继承 (extends ZQAgent) |
| Tools | 6 (bash + 5 file) | 7 (bash + 5 file + todo) |
| Todo | (无) | TodoManager + TodoDefinition |
| 钩子 | (无) | `onToolExecution()` nag reminder |
| 静态字段 | (无) | `static TodoManager todoManager` |
| 约束 | (无) | 最多 20 项, 同时仅 1 个 in_progress |
| 版本追踪 | (无) | `version` 字段驱动 nag 判断 |
