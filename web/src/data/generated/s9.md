# s09: Agent Teams（团队智能体）

`s01 > s02 > s03 > s04 > s05 > s06 | s07 > s08 > [ s09 ] s10 > s11 > s12`

> *"When the task is too big for one, delegate to teammates"* -- persistent teammates + async mailboxes

## 问题

子智能体（s04）是一次性的：生成→工作→返回摘要→销毁。没有身份，没有跨任务记忆。后台任务（s08）只能运行shell命令，不能做LLM-guided决策。

真正的团队协作需要：（1）持久化的智能体，生命周期超过单个提示；（2）身份和生命周期管理；（3）智能体之间的通信通道。

## 解决方案

```
队友生命周期:
  spawn -> WORKING -> IDLE -> WORKING -> ... -> SHUTDOWN

通信机制:
  .team/
    config.json           <- team roster + statuses
    inbox/
      alice.jsonl         <- append-only, drain-on-read
      bob.jsonl
      lead.jsonl

              +--------+    send("alice","bob","...")    +--------+
              | alice  | -----------------------------> |  bob   |
              | loop   |    bob.jsonl << {json_line}    |  loop  |
              +--------+                                +--------+
                   ^                                         |
                   |        BUS.read_inbox("alice")          |
                   +---- alice.jsonl -> read + drain ---------+
```

## Java 实现原理

### 1. TeammateManager - 队友管理器

```java
public class TeammateManager {
    private final Path teamDir;
    private final Path configPath;
    private TeamConfig config;
    private final Map<String, Thread> threads;

    public String spawn(String name, String role, String prompt) {
        TeamMember member = new TeamMember(name, role, "working");
        config.getMembers().add(member);
        saveConfig();

        // 启动队友线程
        Thread thread = new Thread(() -> teammateLoop(name, role, prompt));
        thread.setDaemon(true);
        threads.put(name, thread);
        thread.start();

        return "Spawned '" + name + "' (role: " + role + ")";
    }
}
```

### 2. MessageBus - 消息总线

```java
public class MessageBus {
    public String send(String sender, String to, String content, String msgType, Map<String, Object> extra) {
        TeamMessage msg = new TeamMessage(msgType, sender, content);
        Map<String, Object> msgMap = new LinkedHashMap<>();
        msgMap.put("type", msg.getType());
        msgMap.put("from", msg.getFrom());
        msgMap.put("content", msg.getContent());
        msgMap.put("timestamp", msg.getTimestamp());

        Path inboxPath = inboxDir.resolve(to + ".jsonl");
        try (BufferedWriter writer = Files.newBufferedWriter(inboxPath, 
                StandardOpenOption.CREATE, StandardOpenOption.APPEND)) {
            String jsonLine = objectMapper.writeValueAsString(msgMap);
            writer.write(jsonLine);
            writer.newLine();
            return "Sent " + msgType + " to " + to;
        }
    }

    public List<Map<String, Object>> readInbox(String name) {
        Path inboxPath = inboxDir.resolve(name + ".jsonl");
        List<String> lines = Files.readAllLines(inboxPath);
        List<Map<String, Object>> messages = new ArrayList<>();

        for (String line : lines) {
            if (line != null && !line.trim().isEmpty()) {
                JsonNode jsonNode = objectMapper.readTree(line);
                Map<String, Object> map = objectMapper.convertValue(jsonNode, LinkedHashMap.class);
                messages.add(map);
            }
        }

        // 清空收件箱（drain）
        Files.write(inboxPath, new byte[0]);
        return messages;
    }
}
```

### 3. 队友主循环

```java
private void teammateLoop(String name, String role, String prompt) {
    String systemPrompt = String.format(
        "You are '%s', role: %s, at %s. Use send_message to communicate.",
        name, role, workdir
    );

    List<MessageParam> messages = new ArrayList<>();
    messages.add(MessageParam.builder()
        .role(MessageParam.Role.USER)
        .content(prompt)
        .build());

    for (int i = 0; i < 50; i++) {
        // 检查收件箱
        List<Map<String, Object>> inbox = messageBus.readInbox(name);
        if (!inbox.isEmpty()) {
            String inboxContent = objectMapper.writeValueAsString(inbox);
            messages.add(MessageParam.builder()
                .role(MessageParam.Role.USER)
                .content("<inbox>\n" + inboxContent + "\n</inbox>")
                .build());
        }

        // 调用LLM
        Message response = client.messages().create(...);

        if (response.stopReason() != Message.StopReason.TOOL_USES) {
            break;
        }

        // 执行工具
        for (ContentBlock block : response.content()) {
            if (block.type() == ContentBlock.Type.TOOL_USE) {
                String output = executeTool(name, block.name(), block.input());
                // 添加工具结果...
            }
        }
    }

    // 标记为空闲
    TeamMember member = findMember(name);
    if (member != null) {
        member.setStatus("idle");
        saveConfig();
    }
}
```

### 4. Lead智能体注入收件箱

```java
@Override
protected Message chatMessage(List<MessageParam> messageParams) {
    // 在每次LLM调用前检查lead的收件箱
    injectInboxMessages(messageParams);
    return super.chatMessage(messageParams);
}

private void injectInboxMessages(List<MessageParam> messageParams) {
    List<Map<String, Object>> inbox = messageBus.readInbox("lead");
    if (!inbox.isEmpty()) {
        String inboxContent = objectMapper.writeValueAsString(inbox);

        messageParams.add(MessageParam.builder()
            .role(MessageParam.Role.USER)
            .content("<inbox>\n" + inboxContent + "\n</inbox>")
            .build());

        messageParams.add(MessageParam.builder()
            .role(MessageParam.Role.ASSISTANT)
            .content("Noted inbox messages.")
            .build());
    }
}
```

## 核心优势

1. **持久化队友**：队友配置保存在config.json，可复用
2. **异步通信**：基于JSONL文件的收件箱，支持异步消息传递
3. **状态管理**：完整的队友生命周期管理
4. **线程安全**：使用并发容器保证线程安全
5. **独立运行**：每个队友在独立线程中运行，互不干扰

## 执行示例

**团队协作场景**：

```
用户: 生成3个队友：alice(coder)、bob(tester)、charlie(reviewer)
AI:  [spawn_teammate alice] -> Spawned 'alice' (role: coder)
     [spawn_teammate bob] -> Spawned 'bob' (role: tester)
     [spawn_teammate charlie] -> Spawned 'charlie' (role: reviewer)

[3个队友在独立线程中运行]

用户: 告诉alice编写排序函数
AI:  [send_message to="alice", content="Write a sorting function"]

[alice收到消息，开始编写代码]

用户: 告诉bob等待alice完成后测试代码
AI:  [send_message to="bob", content="Wait for alice then test the code"]

[bob收到消息，等待...]

用户: 告诉charlie审查代码
AI:  [send_message to="charlie", content="Review the code when ready"]

[charlie收到消息，等待...]
```

## 相对于 s08 的变更

| 组件 | s08 (Background Tasks) | s09 (Agent Teams) |
|-----|----------------------|-------------------|
| Tools | 6 | 9 (+spawn/send/read_inbox/broadcast) |
| Agents | Single | Lead + N teammates |
| Persistence | None | config.json + JSONL inboxes |
| Threads | Background commands | Full agent loops per thread |
| Lifecycle | Fire-and-forget | idle -> working -> idle |
| Communication | None | message + broadcast |

## 核心特性

- **守护线程**：使用daemon线程，JVM退出时自动终止
- **JSONL存储**：收件箱使用JSONL格式，便于追加和读取
- **自动清理**：readInbox()自动清空收件箱（drain模式）
- **配置持久化**：队友配置保存在config.json
- **状态同步**：实时更新队友状态到配置文件
