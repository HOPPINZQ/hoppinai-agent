# s10: Team Protocols（团队协议）

`s01 > s02 > s03 > s04 > s05 > s06 | s07 > s08 > s09 > [ s10 ] s11 > s12`

> *"Teammates need shared communication rules"* -- one request-response pattern drives all negotiation

## 问题

在 s09 中，队友可以工作和通信，但缺乏结构化的协调机制：

**关闭问题**：直接杀死线程会导致文件只写入一半、config.json状态过时。需要握手机制：lead请求，teammate批准（完成并退出）或拒绝（继续工作）。

**计划审批**：当lead说"重构auth模块"时，teammate立即开始工作。对于高风险变更，lead应该先审查计划。

两者共享相同的结构：一方发送带唯一ID的请求，另一方引用该ID进行响应。

## 解决方案

```
Shutdown Protocol            Plan Approval Protocol
==================           ======================

Lead             Teammate    Teammate           Lead
 |                 |           |                 |
 |--shutdown_req-->|           |--plan_req------>|
 | {req_id:"abc"}  |           | {req_id:"xyz"}  |
 |                 |           |                 |
 |<--shutdown_resp-|           |<--plan_resp-----|
 | {req_id:"abc",  |           | {req_id:"xyz",  |
 |  approve:true}  |           |  approve:true}  |

共享FSM:
  [pending] --approve--> [approved]
  [pending] --reject---> [rejected]

Trackers:
  shutdown_requests = {req_id: {target, status}}
  plan_requests     = {req_id: {from, plan, status}}
```

## Java 实现原理

### 1. ProtocolTracker - 协议跟踪器

```java
public class ProtocolTracker {
    private final Map<String, RequestTracker> shutdownRequests;
    private final Map<String, RequestTracker> planRequests;

    public String generateRequestId() {
        return UUID.randomUUID().toString().substring(0, 8);
    }

    public RequestTracker createShutdownRequest(String teammate) {
        String requestId = generateRequestId();
        RequestTracker tracker = new RequestTracker(requestId, teammate);
        tracker.setStatus("pending");
        shutdownRequests.put(requestId, tracker);
        return tracker;
    }
}
```

### 2. Shutdown Protocol

**Lead发起关闭请求**：

```java
private static ToolDefinition createShutdownRequestTool(MessageBus mb, ProtocolTracker pt) {
    return new ToolDefinition(
        "shutdown_request",
        "Request a teammate to shut down gracefully.",
        ToolDefinition.createInputSchema(
            Map.of("teammate", createProperty("string", "Teammate name to shutdown")),
            List.of("teammate")
        ),
        ShutdownRequestInput.class,
        input -> {
            ShutdownRequestInput req = (ShutdownRequestInput) input;
            RequestTracker tracker = pt.createShutdownRequest(req.getTeammate());
            
            mb.send("lead", req.getTeammate(), "Please shut down gracefully.", 
                "shutdown_request", Map.of("request_id", tracker.getRequestId()));
            
            return "Shutdown request " + tracker.getRequestId() + " sent (status: pending)";
        }
    );
}
```

**Teammate响应关闭请求**：

```java
if ("shutdown_response".equals(toolName)) {
    Map<String, Object> params = (Map<String, Object>) input;
    String reqId = (String) params.get("request_id");
    Boolean approve = (Boolean) params.get("approve");
    
    protocolTracker.updateShutdownStatus(reqId, approve);
    
    messageBus.send(sender, "lead", reason, "shutdown_response", 
        Map.of("request_id", reqId, "approve", approve));
    
    if (approve) {
        shouldExit = true;  // 优雅退出
    }
    
    return "Shutdown " + (approve ? "approved" : "rejected");
}
```

### 3. Plan Approval Protocol

**Teammate提交计划**：

```java
if ("plan_approval".equals(toolName)) {
    Map<String, Object> params = (Map<String, Object>) input;
    String planText = (String) params.get("plan");
    
    RequestTracker tracker = protocolTracker.createPlanRequest(sender, planText);
    
    messageBus.send(sender, "lead", planText, "plan_approval_response",
        Map.of("request_id", tracker.getRequestId(), "plan", planText));
    
    return "Plan submitted (request_id=" + tracker.getRequestId() + "). Waiting for approval.";
}
```

**Lead审批计划**：

```java
private static ToolDefinition createPlanApprovalTool(MessageBus mb, ProtocolTracker pt) {
    return new ToolDefinition(
        "plan_approval",
        "Approve or reject a teammate plan.",
        ToolDefinition.createInputSchema(
            Map.of(
                "requestId", createProperty("string", "Request ID of the plan"),
                "approve", createProperty("boolean", "True to approve"),
                "feedback", createProperty("string", "Optional feedback")
            ),
            List.of("requestId", "approve")
        ),
        PlanApprovalInput.class,
        input -> {
            PlanApprovalInput plan = (PlanApprovalInput) input;
            RequestTracker tracker = pt.getPlanRequest(plan.getRequestId());
            
            pt.updatePlanStatus(plan.getRequestId(), plan.isApprove());
            
            mb.send("lead", tracker.getFrom(), feedback, "plan_approval_response",
                Map.of("request_id", plan.getRequestId(), "approve", plan.isApprove()));
            
            return "Plan " + (plan.isApprove() ? "approved" : "rejected");
        }
    );
}
```

## 核心优势

1. **优雅关闭**：避免数据丢失和状态不一致
2. **计划审批**：高风险变更前进行审查
3. **关联机制**：request_id确保请求响应匹配
4. **状态追踪**：实时查看协议状态
5. **统一模式**：两种协议共享相同的FSM

## 执行示例

**Shutdown Protocol流程**：

```
用户: 生成alice，然后请求她关闭
AI:  [spawn_teammate name="alice" role="coder"] -> Spawned 'alice'
    [shutdown_request teammate="alice"] -> Shutdown request abc123 sent

[alice收到shutdown_request消息]
alice: [shutdown_response requestId="abc123" approve=true]
-> status变为shutdown，优雅退出
```

**Plan Approval流程**：

```
用户: 告诉bob提交重构计划
AI:  [send_message to="bob" content="Please submit a refactoring plan"]

[bob收到消息]
bob: [plan_approval plan="Refactor auth module: 1) Extract validation 2) Add tests"]
-> Plan submitted (request_id=xyz789). Waiting for approval.

用户: 审批计划
AI:  [plan_approval requestId="xyz789" approve=true feedback="Looks good"]
-> Plan approved for 'bob'

[bob收到approval，开始执行]
```

## 相对于 s09 的变更

| 组件 | s09 (Agent Teams) | s10 (Team Protocols) |
|-----|-------------------|----------------------|
| Tools | 9 | 12 (+shutdown_req/resp +plan) |
| Shutdown | Natural exit only | Request-response handshake |
| Plan gating | None | Submit/review with approval |
| Correlation | None | request_id per request |
| FSM | None | pending -> approved/rejected |

## 核心特性

- **唯一ID**：使用UUID生成唯一的request_id
- **线程安全**：使用ConcurrentHashMap保证线程安全
- **状态管理**：pending -> approved/rejected FSM
- **可追溯**：保留请求历史，支持审计
- **双向通信**：Lead和Teammate都可发起协议
