# s07: Task System（任务系统）

`s01 > s02 > s03 > s04 > s05 > s06 > [ s07 ] | s08 > s09 > s10 > s11 > s12`

> *"将大目标分解为小任务，排序，持久化到磁盘"*

## 问题

s03 的 TodoManager 是内存中的平面检查清单：没有顺序、没有依赖、除了完成/未完成外没有其他状态。真实的目标有结构 —— 任务 B 依赖任务 A，任务 C 和 D 可以并行运行，任务 E 等待 C 和 D 都完成。

没有明确的关系，智能体无法判断：
- 哪些任务可以执行？
- 哪些任务被阻塞？
- 哪些任务可以并行？

而且因为列表只存在于内存中，上下文压缩（s06）会将其清空。

## 解决方案

将检查清单提升为**持久化到磁盘的 DAG 任务图**。每个任务是一个 JSON 文件，包含状态、依赖（`blockedBy`）和被依赖（`blocks`）。任务图在任何时刻都能回答三个问题：

- **哪些任务准备就绪？** — `pending` 状态且 `blockedBy` 为空
- **哪些任务被阻塞？** — `blockedBy` 非空，等待未完成依赖
- **哪些任务已完成？** — `completed` 状态，完成时自动解除依赖

```
                任务图 (DAG)
┌──────────────────────────────────────────────┐
│                                              │
│  task_1 (completed)                          │
│     ├── task_2 (pending, blockedBy: [])      │  ← 可执行
│     └── task_3 (pending, blockedBy: [])      │  ← 可执行
│           ├── task_4 (blocked by: [2, 3])    │  ← 等待 2 和 3
│           └── ...                            │
│                                              │
│  存储格式: .tasks/task_{id}.json             │
│  字段: id, subject, description, status,     │
│        blockedBy, blocks, owner              │
└──────────────────────────────────────────────┘
```

## 工作原理

### 1. TaskManager — 任务管理器

```java
public class TaskManager {
    private final Path tasksDir;  // .tasks/ 目录
    private int nextId;           // 自增 ID

    public TaskManager() {
        this.tasksDir = Paths.get(AIConstants.ROOT, ".tasks");
        this.nextId = findMaxId() + 1;  // 扫描已有文件，保证 ID 连续
    }
}
```

**核心方法：**

```java
// 创建任务 — 自增 ID，默认 pending
public String createTask(String subject, String description) {
    TaskInput task = new TaskInput();
    task.setId(nextId++);
    task.setStatus("pending");
    saveTask(task);
    return taskToJson(task);
}

// 更新任务 — 完成时自动清除依赖
public String updateTask(int taskId, String status,
                         List<Integer> addBlockedBy,
                         List<Integer> addBlocks) {
    TaskInput task = loadTask(taskId);
    if ("completed".equals(status)) {
        clearDependency(taskId);  // 自动解锁后续任务
    }
    // addBlocks 时双向同步被阻塞任务的 blockedBy
    if (addBlocks != null) {
        for (Integer blockedId : addBlocks) {
            TaskInput blockedTask = loadTask(blockedId);
            blockedTask.getBlockedBy().add(taskId);
            saveTask(blockedTask);
        }
    }
    saveTask(task);
    return taskToJson(task);
}

// 清除依赖 — 遍历所有任务，移除已完成任务的 ID
private void clearDependency(int completedId) {
    Files.list(tasksDir).forEach(p -> {
        TaskInput task = loadTaskFromFile(p);
        if (task.getBlockedBy().contains(completedId)) {
            task.getBlockedBy().remove(Integer.valueOf(completedId));
            saveTask(task);
        }
    });
}
```

## 相对于 s06 的变更

| 组件 | s06 | s07 |
|------|-----|-----|
| 工具数 | 10 | 14 (+4 任务工具) |
| 任务模型 | 内存平面清单（TodoManager） | DAG 任务图（磁盘持久化） |
| 依赖关系 | 无 | blockedBy + blocks |
| 状态管理 | done/not | pending → in_progress → completed |
| 持久化 | 压缩后丢失 | `.tasks/` 磁盘文件，压缩后保留 |

| Agent 字段 | 4 个 public static | 5 个 public static (+taskManager) |
