# s05: Skills（技能加载）

`s01 > s02 > s03 > s04 > [ s05 ] s06 | s07 > s08 > s09 > s10 > s11 > s12`

> *"用到什么知识，临时加载什么知识"* -- 通过 tool_result 注入，不塞 system prompt

## 问题

你希望智能体遵循特定领域的工作流：git 约定、测试模式、代码审查清单。全塞进系统提示太浪费 -- 10 个技能，每个 2000 token，就是 20,000 token，大部分跟当前任务毫无关系。

## 解决方案

```
系统提示 (Layer 1 -- 始终存在):
+--------------------------------------+
| 你是一个带有待办事项列表的编程智能体  |
| 可用技能：                           |
|   - git: Git工作流助手和最佳实践      |  ~100 tokens/skill
|   - test: 测试最佳实践和模式          |
+--------------------------------------+

当模型调用 load_skill("git") 时:
+--------------------------------------+
| tool_result (Layer 2 -- 按需加载):    |
| <skill name="git">                   |
|   完整的 Git 工作流说明...           |  ~2000 tokens
|   步骤 1: ...                         |
| </skill>                             |
+--------------------------------------+
```

第一层：系统提示中放技能名称（低成本）。第二层：tool_result 中按需放完整内容。

## 工作原理

### 1. 技能文件结构

每个技能是一个目录，包含 `SKILL.md` 文件和 YAML frontmatter：

```
src/main/resources/skills/
  code-review/
    SKILL.md    # ---\n name: code-review\n description: 审查代码\n ---\n ...
  git/
    SKILL.md    # ---\n name: git\n description: Git工作流\n ---\n ...
  test/
    SKILL.md    # ---\n name: test\n description: 测试实践\n ---\n ...
```

### 2. SkillLoader 实现

```java
public class SkillLoader {
    private final Map<String, String> skills = new HashMap<>();

    public SkillLoader() {
        loadSkills();  // 递归扫描所有 SKILL.md 文件
    }

    // Layer 1: 获取所有技能的简短描述（用于系统提示）
    public String getDescriptions() {
        StringBuilder sb = new StringBuilder();
        skills.forEach((name, body) -> {
            String desc = "-";
            if (body.contains("description=\"")) {
                int start = body.indexOf("description=\"") + 13;
                int end = body.indexOf("\"", start);
                if (end > start) desc = body.substring(start, end);
            }
            sb.append(String.format("  - %s: %s\n", name, desc));
        });
        return sb.toString().trim();
    }

    // Layer 2: 获取指定技能的完整内容（用于 tool_result）
    public String getSkill(String skillName) {
        if (!skills.containsKey(skillName)) {
            return "错误：未知技能 '" + skillName + "'。可用技能："
                + String.join(", ", skills.keySet());
        }
        return skills.get(skillName);
    }
}
```

### 3. YAML Frontmatter 解析

```java
private void parseSkillContent(String entryName, String content) {
    Pattern pattern = Pattern.compile("^---\\n(.*?)\\n---\\n(.*)", Pattern.DOTALL);
    Matcher matcher = pattern.matcher(content);

    String name = entryName;
    String description = "";
    String body = content;

    if (matcher.find()) {
        String meta = matcher.group(1);
        body = matcher.group(2).trim();

        for (String line : meta.split("\n")) {
            String[] parts = line.split(":", 2);
            if (parts[0].trim().equals("name"))
                name = parts[1].trim();
            else if (parts[0].trim().equals("description"))
                description = parts[1].trim();
        }
    }

    skills.put(name, String.format(
        "<skill name=\"%s\" description=\"%s\">\n%s\n</skill>",
        name, description, body));
}
```

### 4. load_skill 工具定义

```java
public static ToolDefinition SkillsDefinition = new ToolDefinition(
    "load_skill",
    "加载指定的技能（工具集合）。可用技能："
        + String.join(", ", Agent05.skillLoader.getAvailableSkills()),
    createInputSchema(
        Map.of("skill_name", createProperty("string", "要加载的技能名称")),
        List.of("skill_name")
    ),
    LoadSkillInput.class,
    Tools::skillLoad
);
```

注意 `Agent05.skillLoader` -- 静态字段桥接，ToolDefinition 中的静态工具定义通过它访问 SkillLoader 实例。

### 5. 双模式加载

SkillLoader 支持两种加载环境：

```java
private void loadSkills() {
    URL resourceUrl = getClass().getClassLoader().getResource(SKILL_PATH);
    String protocol = resourceUrl.getProtocol();

    if ("file".equals(protocol)) {
        loadSkillsFromFileSystem(resourceUrl);  // 开发时：文件系统
    } else if ("jar".equals(protocol)) {
        loadSkillsFromJar(resourceUrl);          // 打包后：从 JAR 读取
    }
}
```
