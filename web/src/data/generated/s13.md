# s13: MCP 工具集成（Model Context Protocol）

`s01 > s02 > s03 > s04 > s05 > s06 > s07 > s08 > [ s09 ] > s10 > s11 > s12`

> *"让智能体接入外部世界"* — 通过 MCP 协议，AI 不再局限于本地工具，可以调用任意 MCP 服务器提供的专业能力

## 问题

s01~s08 的工具都是**硬编码**的 6 个静态工具（bash、read_file、write_file、edit_file、list_files、content_search）。如果需要接入外部系统（数据库、论坛 API、搜索引擎、专业计算引擎），就得在 `ToolDefinition` 里一行行写 Java 代码，重新编译部署。每接入一个新系统，都是一次开发周期。

**核心矛盾：**
- ❌ 工具集固定，无法动态扩展
- ❌ 接入外部系统需要改代码、重新编译
- ❌ 不同系统有各自的认证、传输、协议细节
- ❌ 没有标准化的工具发现和调用机制

## 解决方案

引入 **MCP（Model Context Protocol）** — Anthropic 提出的开放协议，标准化 AI 与外部工具的通信。Agent13 只需编辑 `mcp.json` 配置文件，即可动态加载任意 MCP 服务器提供的工具，无需改代码。

## 工作原理

### 1. McpConfigLoader — 配置解析与传输类型自动检测

从 `resources/mcp.json` 读取配置，自动判断传输方式：

```java
public List<McpSetting> loadMcpSettings() {
    InputStream inputStream = getClass().getResourceAsStream("/mcp.json");
    JsonNode rootNode = objectMapper.readTree(inputStream);
    JsonNode serversNode = rootNode.get("mcpServers");  // 对象，非数组

    serversNode.fields().forEachRemaining(entry -> {
        String serverName = entry.getKey();       // 键 = 服务器名称
        JsonNode serverConfig = entry.getValue();  // 值 = 服务器配置

        if (!isEnabled(serverConfig)) return;      // 支持 "enabled" 字段

        McpSetting setting = parseMcpSetting(serverName, serverConfig);
        settings.add(setting);
    });
    return settings;
}
```

**自动传输检测规则：**

```java
if (serverConfig.has("command")) {
    // STDIO — 启动本地进程，通过标准输入输出通信
    builder.transportType(TransportType.STDIO);
    configureStdio(builder, serverConfig);  // 解析 command + args + env
} else if (serverConfig.has("url")) {
    String url = serverConfig.get("url").asText();
    if (url.contains("/sse")) {
        builder.transportType(TransportType.SSE);          // 0.9.0 已废弃
    } else {
        builder.transportType(TransportType.Streamable_HTTP);  // 推荐的 HTTP 方式
    }
    configureHttp(builder, serverConfig);  // 解析 url + headers + authorization
}
```

**mcp.json 配置格式（Claude 标准格式）：**


### 2. MCPAgent — 客户端生命周期管理

根据传输类型创建对应的 MCP 客户端，支持同步/异步两种模式：

```java
public void initializeClient() {
    mcpSettings.forEach(setting -> {
        McpClientTransport transport;
        switch (setting.getTransportType()) {
            case STDIO:
                transport = new StdioClientTransport(setting.getServerParameters());
                break;
            case Streamable_HTTP:
                transport = new WebFluxSseClientTransport(
                    WebClient.builder()
                        .baseUrl(setting.getSseUrl())
                        .defaultHeaders(h -> h.add("Authorization", setting.getAuthorization())),
                    OBJECT_MAPPER, McpSetting.sseEndpoint);
                break;
            default:
                throw new RuntimeException("不支持的传输方式");
        }

        if (setting.isSync()) {
            McpSyncClient syncClient = McpClient.sync(transport)
                .requestTimeout(setting.getTimeout())  // 默认 10 秒
                .build();
            syncClient.initialize();
            setting.setMcpSyncClient(syncClient);
        }
        // 异步模式类似，使用 McpClient.async()...
    });
}
```

### 3. McpLoader — MCP Schema → Anthropic ToolDefinition 桥接

核心转换逻辑：将 MCP 协议的工具描述转为 Anthropic SDK 可用的格式：

```java
public List<ToolDefinition> loadTools() {
    List<ToolDefinition> tools = new ArrayList<>();
    for (McpSetting setting : mcpSettings) {
        McpSchema.ListToolsResult mcpTools = setting.isSync()
            ? setting.getMcpSyncClient().listTools()
            : setting.getMcpAsyncClient().listTools().block();

        for (McpSchema.Tool mcpTool : mcpTools.tools()) {
            ToolDefinition toolDef = convertToToolDefinition(mcpTool, setting);
            tools.add(toolDef);
        }
    }
    return tools;
}
```

**Schema 转换（DeepSeek 兼容版本）：**

```java
private ToolDefinition convertToToolDefinition(McpSchema.Tool mcpTool, McpSetting setting) {
    McpSchema.JsonSchema jsonSchema = mcpTool.inputSchema();
    ObjectNode propertiesNode = OBJECT_MAPPER.valueToTree(jsonSchema.properties());
    List<String> requiredList = jsonSchema.required();

    Tool.InputSchema.Builder schemaBuilder = Tool.InputSchema.builder()
        .properties(com.anthropic.core.JsonValue.fromJsonNode(propertiesNode));
    if (requiredList != null && !requiredList.isEmpty()) {
        schemaBuilder.required(requiredList);
    }

    return ToolDefinition.builder()
        .name(mcpTool.name())
        .description(mcpTool.description())
        .inputSchema(schemaBuilder.build())
        .function(createExecuteFunction(setting))
        .build();
}
```

### 4. createExecuteFunction — MCP 工具执行闭包

每个 MCP 工具的执行函数是一个闭包，捕获了对应的 `McpSetting`，通过 JSON 中转调用：

```java
private Function<String, String> createExecuteFunction(McpSetting setting) {
    return new Function<String, String>() {
        @Override
        public String apply(String inputJson) {
            AtomicReference<String> result = new AtomicReference<>("");

            // 1. 解析输入 — 从 JSON 中提取 tool_name 和 input
            JsonNode jsonNode = OBJECT_MAPPER.readTree(inputJson);
            String toolName = jsonNode.get("tool_name").asText();
            Map<String, Object> arguments = OBJECT_MAPPER.convertValue(
                jsonNode.get("input"),
                new TypeReference<Map<String, Object>>() {}
            );

            // 2. 调用 MCP 服务器上的工具
            McpSchema.CallToolResult callResult = setting.isSync()
                ? setting.getMcpSyncClient().callTool(
                    new McpSchema.CallToolRequest(toolName, arguments))
                : setting.getMcpAsyncClient().callTool(
                    new McpSchema.CallToolRequest(toolName, arguments)).block();

            // 3. 提取文本结果
            if (callResult != null && callResult.content() != null) {
                for (McpSchema.Content content : callResult.content()) {
                    if (content instanceof McpSchema.TextContent) {
                        result.set(((McpSchema.TextContent) content).text());
                    }
                }
            }
            return result.get();
        }
    };
}
```

### 5. Agent13 — 组合模式启动

Agent13 使用**组合模式**（与 Agent01 相同），不继承 ZQAgent：

```java
public class Agent13 {
    public static void main(String[] args) {
        AnthropicClient client = AnthropicOkHttpClient.builder()
            .apiKey(API_KEY).baseUrl(BASE_URL).build();

        // 1. 加载 MCP 配置
        List<McpSetting> settings = new McpConfigLoader().loadMcpSettings();

        // 2. 初始化 MCP 客户端
        MCPAgent mcpAgent = new MCPAgent(settings);
        mcpAgent.initializeClient();

        // 3. 加载 MCP 工具
        McpLoader mcpLoader = new McpLoader(settings);
        System.out.println(mcpLoader.getServerInfo());
        System.out.println(mcpLoader.getToolDescriptions());

        // 4. 组装工具列表：6 内置 + N MCP
        List<ToolDefinition> tools = new ArrayList<>();
        tools.add(BashDefinition);
        tools.add(ReadFileDefinition);
        tools.add(WriteFileDefinition);
        tools.add(EditFileDefinition);
        tools.add(ListFilesDefinition);
        tools.add(ContentSearchDefinition);
        tools.addAll(mcpLoader.loadTools());

        // 5. 创建 ZQAgent 并注入动态系统提示
        ZQAgent agent = new ZQAgent(client, MODEL, tools);
        agent.setSystemPrompt(buildSystemPrompt(mcpLoader));
        agent.run();

        // 6. 优雅关闭 MCP 连接
        mcpAgent.closeClient();
    }
}
```

### 6. McpSetting — 配置数据模型

```java
public class McpSetting {
    public static final String sseEndpoint = "/sse";

    private String mcpId;           // UUID
    private String name;            // 服务器名称（mcp.json 中的 key）
    private String description;     // 描述
    private ClientType clientType;  // SYNC / ASYNC
    private TransportType transportType;  // STDIO / SSE / Streamable_HTTP
    private McpSyncClient mcpSyncClient;  // 同步客户端实例
    private McpAsyncClient mcpAsyncClient; // 异步客户端实例
    private Duration timeout;       // 默认 10 秒
    private ServerParameters serverParameters;  // STDIO 配置
    private String sseUrl;          // HTTP/SSE 地址
    private String authorization;   // 认证令牌
    private Map<String, String> headers;  // 自定义请求头
    private List<McpSchema.Root> roots;   // 客户端能力配置
}
```

## 相对于 s08 的变更

| 组件 | s08 | s13 |
|------|-----|-----|
| 架构模式 | 继承 ZQAgent（Agent08 extends ZQAgent） | 组合模式（不继承，创建 ZQAgent 实例） |
| 工具数量 | 16（固定硬编码） | 6 + N（动态加载 MCP 工具） |
| 工具扩展 | 修改 ToolDefinition.java，重新编译 | 编辑 mcp.json，无需改代码 |
| 外部系统集成 | 不支持 | 通过 MCP 协议标准化接入 |
| 传输方式 | 无 | STDIO / SSE / Streamable_HTTP |
| 客户端类型 | 无 | 同步（SYNC）/ 异步（ASYNC） |
| 配置方式 | 代码内硬编码 | mcp.json 配置文件 |
| 工具发现 | 无 | listTools() 自动发现 |
| 资源管理 | 无 | listResources() + readResource() |
| 提示词模板 | 无 | listPrompts() |
| 系统提示 | 静态字符串 | 动态生成（含 MCP 工具描述） |
| 后台任务 | BackgroundManager + CachedThreadPool | 无（回归 s01 级别基础能力） |
| 子智能体 | SubAgentTool | 无 |
| Todo 管理 | TodoManager | 无 |
| Skill 系统 | SkillLoader | 无 |
| 上下文压缩 | ContextCompactor | 无 |
| 新增类 | — | McpLoader, MCPAgent, McpConfigLoader, McpSetting |
