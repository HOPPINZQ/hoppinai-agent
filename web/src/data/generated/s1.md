# s01: The Agent Loop (智能体循环)

`[ s01 ] s02 > s03 > s04 > s05 > s06 | s07 > s08 > s09 > s10 > s11 > s12`

> *"One loop & Tool is all you need"* -- 一个工具 + 一个循环 = 一个智能体。

## 问题

语言模型能推理代码, 但碰不到真实世界 -- 不能读文件、跑测试、看报错。没有循环, 每次工具调用你都得手动把结果粘回去。你自己就是那个循环。

## 解决方案

```
+--------+      +-------+      +---------+
|  User  | ---> |  LLM  | ---> |  Tool   |
| prompt |      |       |      | execute |
+--------+      +---+---+      +----+----+
                    ^                |
                    |   tool_result  |
                    +----------------+
                    (loop until stop_reason != "tool_use")
```

一个退出条件控制整个流程。循环持续运行, 直到模型不再调用工具。

## 工作原理

### 1. 用户输入处理

将用户的输入添加到消息列表中。在Java中,我们使用`MessageParam`对象来表示消息:

```java
MessageParam userMessage = MessageParam.builder()
        .role(MessageParam.Role.USER)
        .content(userInput)
        .build();
messageParams.add(userMessage);
```

**代码解释**:
- `MessageParam.builder()`: 使用建造者模式创建消息参数
- `role(MessageParam.Role.USER)`: 设置消息角色为用户
- `content(userInput)`: 设置消息内容为用户输入的文本
- `messageParams`: 累积式消息列表,保存整个对话历史

### 2. 发送请求到LLM

将消息和工具定义一起发给LLM:

```java
protected Message chatMessage(List<MessageParam> messageParams){
    List<ToolUnion> anthropicTools = new ArrayList<>();
    for (ToolDefinition tool : tools) {
        anthropicTools.add(ToolUnion.ofTool(
                Tool.builder()
                        .name(tool.getName())
                        .description(tool.getDescription())
                        .inputSchema(tool.getInputSchema())
                        .build()
        ));
    }

    MessageCreateParams.Builder messageBuilder = MessageCreateParams.builder()
            .model(model)
            .messages(messageParams)
            .tools(anthropicTools);

    if (systemPrompt != null && !systemPrompt.isEmpty()) {
        messageBuilder.system(systemPrompt);
    }
    messageBuilder.maxTokens(MAX_TOKENS);
    messageBuilder.temperature(TEMPERATURE);

    return client.messages().create(messageBuilder.build());
}
```

**代码解释**:
- `ToolUnion.ofTool()`: 将工具定义转换为Anthropic API所需的格式
- `MessageCreateParams.builder()`: 构建API请求参数
- `client.messages().create()`: 发送请求到Anthropic API
- `systemPrompt`: 可选的系统提示词,用于引导LLM行为
- 返回的`Message`对象包含模型的响应内容和停止原因

### 3. 处理LLM响应并检查工具调用

追加助手响应,检查`stop_reason` -- 如果模型没有调用工具, 结束:

```java
Message message = chatMessage(messageParams);
messageParams.add(message.toParam());

List<ContentBlockParam> toolResults = new ArrayList<>();
boolean hasToolUse = false;

for (ContentBlock content : message.content()) {
    if (content.isText()) {
        Optional<TextBlock> text = content.text();
        String result = text.map(TextBlock::text).orElse("");
        System.out.printf("AI: %s%n", result);
    } else if (content.isToolUse()) {
        hasToolUse = true;
        ToolUseBlock toolUse = content.asToolUse();

        String toolResult = null;
        for (ToolDefinition tool : tools) {
            if (tool.getName().equals(toolUse.name())) {
                try {
                    JsonValue input = toolUse._input();
                    toolResult = invokeTool(tool, input);
                } catch (Exception e) {
                    toolResult = "错误: " + e.getMessage();
                }
                break;
            }
        }

        toolResults.add(ContentBlockParam.ofToolResult(
                ToolResultBlockParam.builder()
                        .toolUseId(toolUse.id())
                        .content(toolResult)
                        .isError(false)
                        .build()
        ));
    }
}

if (!hasToolUse) {
    break;
}
```

**代码解释**:
- `message.content()`: 获取响应内容块列表
- `content.isText()` / `content.isToolUse()`: 判断内容类型
- `hasToolUse`: 标志变量,表示是否需要执行工具
- `invokeTool()`: 匹配工具名并执行,支持两种模式（Function和TypedToolInvoker）
- `ContentBlockParam.ofToolResult()`: 创建工具结果消息块

### 4. 执行工具并返回结果

工具执行后,结果作为user消息追加,回到第2步继续循环:

```java
MessageParam.Content content = MessageParam.Content.ofBlockParams(toolResults);
MessageParam toolResultMessage = MessageParam.builder()
        .role(MessageParam.Role.USER)
        .content(content)
        .build();
messageParams.add(toolResultMessage);
message = chatMessage(messageParams);
messageParams.add(message.toParam());
```

### 完整的智能体循环

将上述步骤组装为完整的ZQAgent类:

```java
public class ZQAgent {
    protected final AnthropicClient client;
    protected final String model;
    private final List<ToolDefinition> tools;
    protected final List<MessageParam> messageParams = new ArrayList<>();
    private String systemPrompt;

    public ZQAgent(AnthropicClient client, String model, List<ToolDefinition> tools) {
        this.client = client;
        this.model = model;
        this.tools = tools;
    }

    public void run() {
        Scanner scanner = new Scanner(System.in);
        System.out.println("开始对话吧");

        while (true) {
            System.out.print("你: ");
            String userInput = scanner.nextLine();
            if (userInput.isEmpty()) continue;

            MessageParam userMessage = MessageParam.builder()
                    .role(MessageParam.Role.USER)
                    .content(userInput)
                    .build();
            messageParams.add(userMessage);

            Message message = chatMessage(messageParams);
            messageParams.add(message.toParam());

            while (true) {
                List<ContentBlockParam> toolResults = new ArrayList<>();
                boolean hasToolUse = false;

                for (ContentBlock content : message.content()) {
                    if (content.isText()) {
                        String result = content.text()
                                .map(TextBlock::text).orElse("");
                        System.out.printf("AI: %s%n", result);
                    } else if (content.isToolUse()) {
                        hasToolUse = true;
                        ToolUseBlock toolUse = content.asToolUse();
                        String toolResult = invokeTool(tool, toolUse._input());
                        toolResults.add(createToolResult(toolUse.id(), toolResult));
                    }
                }

                if (!hasToolUse) break;

                MessageParam toolResultMessage = MessageParam.builder()
                        .role(MessageParam.Role.USER)
                        .content(MessageParam.Content.ofBlockParams(toolResults))
                        .build();
                messageParams.add(toolResultMessage);
                message = chatMessage(messageParams);
                messageParams.add(message.toParam());
            }
        }
    }
}
```

**核心要点**:
- **外层循环**: 处理用户对话,持续运行直到程序退出
- **内层循环**: 处理工具调用,持续运行直到模型不再调用工具
- **消息累积**: `messageParams`保存整个对话历史,包括用户消息、AI响应和工具结果
- **退出条件**: 当`hasToolUse`为false时,说明模型已完成所有工具调用

Agent01 使用**组合模式** -- 不继承ZQAgent, 而是在main方法中创建实例:

```java
public class Agent01 {
    public static void main(String[] args) {
        AnthropicClient client = AnthropicOkHttpClient.builder()
                .apiKey(API_KEY).baseUrl(BASE_URL).build();

        List<ToolDefinition> tools = new ArrayList<>();
        tools.add(BashDefinition);

        ZQAgent agent = new ZQAgent(client, MODEL, tools);
        agent.setSystemPrompt(buildSystemPrompt());
        agent.run();
    }
}
```

不到200行,这就是整个智能体。后面11个章节都在这个循环上叠加机制 -- 循环本身始终不变。

## 变更内容

| 组件 | 之前 | 之后 |
|-----|------|------|
| Agent loop | (无) | `while(true)` + `hasToolUse`标志 |
| Tools | (无) | `bash` (单一工具) |
| Messages | (无) | `List<MessageParam>`累积式消息列表 |
| Control flow | (无) | `hasToolUse == false`退出条件 |
| 类型安全 | 动态类型 | 静态类型,编译时检查 |
